# C4 — Nível 1 (Context)

Pessoas:
- Owner/Agent (Usuário do Painel) — usa o SPA React para operar veículos do seu tenant.
- Superuser (Equipe interna) — usa o mesmo SPA React, mas com funcionalidades administrativas adicionais e privilégios globais.

Sistemas:
- Painel Web (React SPA) — Frontend que consome a API e mostra telas de login, listas e CRUD.  
  * Toda comunicação com a API é resolvida por subdomínio ou header X-Tenant (ver ADR-01).
- API Laravel — Backend REST /api/v1, multitenant, RBAC, validação, auditoria e rate-limit.
Sistemas Externos/Infra:
- Relational DB (RDS) — persistência de tenants, usuários, vehicles e auditoria.
- Object Storage (S3) — armazenamento de imagens de veículos.
- Email/Notificação (ex.: SES/SMTP) — envio de “esqueci a senha”; pode evoluir para uso de fila (ex.: SQS).
- Observabilidade (CloudWatch/ELK) — logs estruturados, métricas, alarmes e tracing básico.

Relações (→):
Usuário → SPA (HTTP/S)

SPA → API (HTTPS, JSON)

API → RDS (SQL)  
API → S3 (upload/listing)  
API → Email (SMTP/SES)  
API → Observabilidade (logs estruturados, métricas, alarmes, tracing)
